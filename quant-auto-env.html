一、开发工具安装与配置

先搭好稳定、可复现的工作台

本部分学习目标：理解每个工具的作用，并完成最基础的运行环境配置。

开发环境可以理解成自动化策略研究的工作台：代码编辑器负责书写和管理脚本，Python 负责运行程序，依赖库负责数据读取、数值计算和图表展示。

  • 代码编辑器：用于管理项目文件，例如 VS Code、PyCharm 或 Jupyter Notebook。
  • Python 环境：用于运行策略研究脚本，建议使用稳定版本，便于安装数据处理与回测工具。
  • 常用依赖库：Pandas 处理表格数据，NumPy 进行数值计算，Matplotlib 用于基础图表展示。

入门阶段不需要追求复杂配置，只要做到能打开代码文件、能运行 Python 程序、能导入基础数据处理工具，就已经具备后续学习条件。

完成本部分后 → 进入运行环境测试 ↓

二、运行环境测试

用最小程序确认环境真的可用

本部分学习目标：学会用最简单的方法，确认环境能正常运行。

工具安装完成后，先不要急着写规则。很多新手遇到的报错并不是规则逻辑问题，而是路径、版本、依赖库或数据文件位置没有配置好。

  • 检查 Python 是否安装成功。
  • 确认编辑器能运行 Python 文件。
  • 确认 Pandas、NumPy 等基础库能正常导入。
  • 尝试读取一小段表格数据，并输出简单结果。
  • 检查输出文件或日志是否能保存到指定位置。

如果测试失败，优先检查文件路径、库是否安装、Python 版本、文件名拼写以及数据文件位置。先排除环境问题，后续才更容易定位规则逻辑问题。

完成本部分后 → 进入项目文件结构 ↓

三、项目文件结构

把数据、代码、配置和结果分开放

本部分学习目标：知道一个策略研究项目里，不同文件分别负责什么。

一个自动化策略研究项目通常不止一个脚本。随着内容变多，如果所有文件混在一起，数据、规则逻辑、回测结果和配置会很快失控。

  • 数据文件：保存历史行情、指标数据或测试数据。
  • 规则文件：保存规则、条件判断和信号生成逻辑。
  • 回测文件：把策略规则放入历史数据中测试并记录结果。
  • 配置文件：保存周期、参数、风险限制和数据路径。
  • 结果文件：保存图表、日志、信号记录和输出报告。

结构清楚，排查问题就更快：数据异常先查数据文件，信号不对先查规则文件，结果异常先查回测和配置。

完成本部分后 → 进入第一个策略程序运行 ↓

四、第一个策略程序运行

跑通从输入到输出的完整流程

本部分学习目标：跑通一个完整小例子，理解程序从输入到输出的基本流程。

环境和文件结构就绪后，就可以运行一个最小规则示例。重点不是复杂代码，而是理解自动化策略研究如何按固定顺序处理数据。

  • 读取数据：打开历史行情或示例表格。
  • 计算指标：例如均线、结果指标或波动范围。
  • 判断条件：例如价格是否高于均线。
  • 记录信号：条件满足时写入结果表。
  • 输出结果：保存表格、图表或日志。

自动化策略研究并不是“自动保证结果”的工具，它本质上是按预设规则处理数据并输出可检查的研究结果。
